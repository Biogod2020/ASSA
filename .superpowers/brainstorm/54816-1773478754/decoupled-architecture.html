ASSA Decoupled Sub-Agent Architecture

Separating Execution (Main Agent) from Evolution (Evolutionary Sub-Agents).

        User->>Main: Next Task

        Mem-->>Main: Hot-Reloaded Knowledge (I've learned a better way!)
        Main->>User: Optimized Execution
        deactivate Main

The Role of Sub-Agents:

Asynchronous Distillation

Heavy reasoning about code diffs and patterns happens in a separate session to save tokens and time.

Automated Verification

A specialized sub-agent acts as a Quality Gate, running tests before any meta-logic is updated.

Self-Triggering

Sub-agents can wake up based on Git Hooks or CRON tasks, evolving the agent while you're away.
