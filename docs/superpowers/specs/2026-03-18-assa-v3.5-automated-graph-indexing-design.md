# Design Spec: ASSA V3.5 (Automated Graph Indexing - "The Weaver")

**Date**: 2026-03-18
**Status**: Approved
**Version**: 3.5-AUTOMATION

## 1. Vision
Transition the ASSA Knowledge Graph from manual JSON maintenance to a fully automated, file-driven system. In V3.5, **Markdown is the Single Source of Truth**. All metadata, dependencies, and trigger domains are defined in Markdown Frontmatter, and a dedicated script generates the high-performance JSON indexes (`graph.json`, `index.json`) used by the agent hooks.

## 2. Standardized Markdown Frontmatter (The "Schema")

To be indexable by the Weaver, all `.md` files in `~/.gemini/assa/LIBRARY/` MUST include the following YAML block:

```yaml
---
id: [UNIQUE-ID]         # Unique identifier (e.g., P-PYTHON-SOTA)
level: [G0-G3]          # Global level (Node weight/priority)
rationale: "[STRING]"   # Concise justification for the rule
depends_on: [ARRAY]     # IDs of prerequisite rules
triggers: [ARRAY]       # Keyword patterns in CWD to activate this rule
evolution_version: 3.5
status: [Active|Deprecated]
supersedes: [OLD-ID]    # Optional: for versioning
---
```

## 3. The Indexing Engine: `rebuildGraph.js`

A Node.js script located at `~/.gemini/assa/scripts/rebuildGraph.js`.

### 3.1 Core Logic
1. **Recursive Scan**: Scans the Global Library for all `.md` files.
2. **Metadata Extraction**: Parses YAML frontmatter using regex (to avoid external dependencies where possible).
3. **Graph Construction**:
   - Generates `graph.json` containing the full rule dictionary.
   - Generates `index.json` mapping `trigger_words` to `RULE_IDs`.
4. **Validation**:
   - **Collision Detection**: Errors if duplicate IDs are found.
   - **Broken Link Detection**: Warns if a `depends_on` ID is missing from the library.
5. **Atomic Write**: Writes both JSON files to the global root.

## 4. The "Auto-Commit" Workflow

### 4.1 Integration with Promoter
1. **Write**: The `Promoter` (Syncer) writes a compliant Markdown file to the library.
2. **Rebuild**: The `Promoter` immediately executes `node ~/.gemini/assa/scripts/rebuildGraph.js`.
3. **Log**: The script outputs a summary of changes (Nodes found, links verified).

### 4.2 Integration with BeforeHook
- The `BeforeHook` remains lean. It only reads the generated `graph.json` and `index.json`. 
- No parsing logic is added to the Hook, maintaining millisecond performance.

## 5. Success Criteria
- [ ] 100% decoupling of rule content and indexing logic.
- [ ] Manual deletion of `graph.json` is recoverable by a single script call.
- [ ] No more "Index Drift" (where a file exists but the index doesn't point to it).
- [ ] Zero manual edits required for any JSON file in the ASSA environment.
