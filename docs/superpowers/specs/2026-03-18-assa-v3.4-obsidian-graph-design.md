# Design Spec: ASSA V3.4 (Obsidian-Style Knowledge Graph)

**Date**: 2026-03-18
**Status**: Implementing
**Version**: 3.4-INTELLIGENCE

## 1. Vision
Transform the ASSA knowledge management system into a hierarchical, link-aware knowledge graph inspired by Obsidian. This design optimizes for both **Intelligence Density** (AI has access to deep context) and **Operational Efficiency** (minimal context overhead during standard operations).

## 2. Hierarchical Node Levels (The "Skeleton" Strategy)

To prevent context explosion, rules are categorized into four levels. The `BeforeAgent` hook performs "Skeleton Loading" (Metadata only) followed by " Meat Loading" (Recursive full content).

| Level | Name | Loading Priority | Scope |
| :--- | :--- | :--- | :--- |
| **L0** | Core Mandates | Permanent | Fundamental personality, safety, and agentic "Soul". |
| **L1** | Foundational Standards | Always for Language | Git conventions, general coding standards, project root rules. |
| **L2** | Domain Specialists | On-Demand (Linked) | Deep tech stacks (React V19, FastAPI, Rust Memory Safety). |
| **L3** | Transient Patterns | On-Demand (Project) | Local project experiences, temporary bug fixes, specific habits. |

## 3. Knowledge Graph Engine (`graph.json`)

`graph.json` acts as the centralized cache for the Obsidian-style graph.

### 3.1 Schema
```json
{
  "version": "3.4",
  "rules": {
    "P-20260318-REFLX": {
      "id": "P-20260318-REFLX",
      "path": "LIBRARY/REFLX.md",
      "level": 1,
      "depends_on": ["P-CORE-001"],
      "rationale": "Ensures the agent is self-aware of its own reflex triggers.",
      "triggers": ["hooks", "refactoring"]
    }
  }
}
```

## 4. Workflows

### 4.1 "Safe Sync" Promotion (Conflict Resolution)
- **Strategy**: Versioning/Deprecation (Superseding).
- **Rule**: When a conflict occurs between a new L3 pattern and an existing L1/L2 rule:
    1. The old rule is marked as `status: deprecated` and `superseded_by: [NEW-ID]`.
    2. The new rule is created with a reference `supersedes: [OLD-ID]`.
- **Human-in-the-Loop**: If the conflicted rule is **L0 or L1**, the Promoter MUST stop and use `ask_user` for confirmation.

### 4.2 "Instant Promotion" (Global Consciousness)
- **Trigger**: Semantic analysis of user intent.
- **Action**: If a user expresses a "Strong Cross-Project Intent" (e.g., "This must be strictly applied to all my future projects"), the agent immediately dispatches the Promoter sub-agent to sync the pattern to L3 Global Library without waiting for `git push`.

### 4.3 Recursive Context Injection (Skeleton -> Meat)
1. **Match**: Hook identifies Seed IDs via CWD or current Task keywords.
2. **Expand**: Hook recursively resolves all `depends_on` links via `graph.json`.
3. **Inject**: Skeleton (IDs + Rationales) is injected first; Full Content (Meat) is injected for the direct Seed IDs.

## 5. Success Criteria
- [ ] BeforeHook overhead remains < 5KB for standard turns.
- [ ] No manual editing of `graph.json` required (fully automated by Promoter).
- [ ] L0/L1 rules are never modified without explicit user approval.
