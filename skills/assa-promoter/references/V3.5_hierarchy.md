# ASSA Knowledge Hierarchy (V3.5)

To maintain a clean and effective knowledge graph, every promoted rule MUST be assigned a `level` in its YAML frontmatter.

## 🌐 Global Levels (G-Series)
These rules reside in `~/.gemini/assa/LIBRARY/` and are shared across all projects.

- **G0 (Global Core)**: Mandates, SOUL, fundamental safety, and core agentic protocols. 
    - *Protection*: Requires explicit user confirmation to modify or supersede.
- **G1 (Global Foundation)**: Universal engineering standards, base language rules, and common CLI tool patterns.
    - *Protection*: Requires explicit user confirmation to modify or supersede.
- **G2 (Global Domain)**: Deep knowledge of specific frameworks, libraries, or platforms (e.g., React, FastAPI, AWS).
- **G3 (Global Fragment)**: Highly specific global utilities, cross-project snippets, or reusable code patterns.

## 📂 Local Levels (L-Series)
These rules are typically found in a project's `.memory/patterns.md` before promotion.

- **L1 (Local Override)**: Project-specific standards that explicitly override G1 or G2 rules for this workspace.
- **L2 (Local Specialist)**: Deep knowledge specific to the current codebase and its unique architectural choices.
- **L3 (Local Transient)**: Temporary patterns or experimental rules being tested before being refined into L2.

## 🧬 Promotion Path
`L3 -> L2 -> [Promoter Skill] -> G2/G1/G0`
